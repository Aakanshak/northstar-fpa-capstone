# Limitations and Next Steps

- Replace synthetic ledgers with billing, CRM, payroll, and ERP integrations with automated reconciliation controls.
- Model contract start/end dates, billing frequency, FX, revenue recognition, and bookings-to-billings timing at customer level.
- Use role-level requisition dates, ramp curves, attrition, commissions, and benefit rates instead of a hiring-pace scalar.
- Replace the three-point forecast with a Monte Carlo distribution for conversion, churn, sales-cycle duration, DSO, and hiring dates.
- Add cohort-specific churn curves, channel attribution, renewal calendars, and product usage leading indicators.
- Add covenant calculations, tax schedules, debt amortization, and formal GAAP revenue-recognition support.
- Replace the formula sensitivity grid with a native Excel What-If Data Table if the model will be maintained only in desktop Excel; the current grid remains transparent and cross-viewer compatible.

## So what?

The current model is decision-ready for a portfolio case, while a production deployment would prioritize source-system controls, probabilistic forecasting, and more granular workforce and revenue-recognition drivers.
